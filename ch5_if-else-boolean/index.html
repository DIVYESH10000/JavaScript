      25.What are Booleans

      1. Data Type: Booleans are a basic data type in JavaScript.
      2. Two Values: Can only be true or false.
      3. ‘true’ is a String not a Boolean.

      26. Comparison Operators

      Equality
        • == Checks value equality.
        • === Checks value and type equality.
      Inequality
        • != Checks value inequality.
        • !== Checks value and type inequality.
      Inequality
        • > Greater than.
        • < Less than.
        • > = Greater than or equal to.
        • < = Less than or equal to.
        ------------------------------------------------------------------
       Note:  Order of comparison operators is less than arithmetic operators

       Examples:--

       console.log(typeof (6 > 9));     // boolean
       console.log(typeof true);        // boolean
       console.log(typeof 'true');      // string
   
   
       console.log(5 == 5);           // true

       console.log(5 == '5');         // true, as == only compares value. Either it will convert int 5 to string 5 or vice versa the compare them.

       console.log(5 == 5.0);         // true
       console.log(5 == 4);           // false
   
       console.log(5==='5.0')           // false as === compares the value as well as typeof value.

      27. if-else


      1. Syntax: Uses if () {} to check a condition.
      2. What is if: Executes block if condition is true, skips if false.
      3. What is else: Executes a block when the if condition is false.
      4. Curly Braces can be omitted for single statements, but not recommended.
      
      5. If-else Ladder: Multiple if and else if blocks; only one executes.
      6. Use Variables: Can store conditions in variables for use in if statements.

      Examples:---

      // Case 1
    console.log("Checking if you can drive: ");
    if (21 > 18) {
      console.log("You can drive")
    }

    // Case 2
    console.log("\n\nChecking if you can drive: ");

    let age = 1;

    if(age >= 18) {
      console.log(`Darling you sure can drive, as your age is ${age} years\nwhich exceeds the min eligibility criteria i.e. 18 years.`);
    }
    else if (age < 18){
      console.log(`Darling you sure can't drive, as your age is ${age} years.\nI suggest riding bicycle. Do enjoy your time while you can.`);
    }
    else if (age >=65) {
      console.log(`Sir/Mam,\nYou can't drive until you get a medical fitness certificate from RTO,\nas you're a senior citizen.`)
    }
    else {
      console.log('get lost')
    }
    
    // For single statements, can skip braces but it's not recommended.
   // else stmt in the end is not mandatory.
   // Note : Only 1 stmt is true in the whole ladder, the moment true stmt executes, rest is skipped.

      28. Logical Operators

      

      1. Types: && (AND), || (OR), ! (NOT)
      2. AND (&&):  All conditions must be true for the result to be true.
      3. OR (||):  Only one condition must be true for the result to be true.
      4. NOT (!): Inverts the Boolean value of a condition.
      5. Lower Priority than Math and Comparison operators.


      Example:--

      let yourAge = 32;

      if(yourAge > 25 && yourAge < 30){
        console.log('You still can become a  under 30.')
      }
      else if (yourAge > 31 || yourAge < 35){
        console.log(`Everyday work like it's the last sday of your life.`)
      }

      30. Truthy and Falsy Values


      1. Falsy Values: 0, null, undefined, false, NaN, "" (empty string)

      2. Truthy Values: All values not listed as falsy.

      3. Used in conditional statements like if.

      4. Non-boolean values are auto-converted in logical operations.

      5. Be explicit in comparisons to avoid unexpected behaviour.

      Example:--

      // When we assign VALUES where BOOLEAN was supposed to be used.

      // let __age = 21;  // Truthy
      // let __age = 0;   // Falsy ------------ 0
      // let __age;       // Falsy ------------ Undefined (When Value not defined.)
      // let __age = '';  // Falsy ------------ Empty String
      // let __age = null;// Falsy ------------ null (When we specifically don't wanna assign value.s)
      let __age = NaN;    // Falsy ------------ NaN (Illogical : 5 / ice-cream)
  
      if (__age) {
        console.log('You have  a valid age');
      } else {
        console.log(`You're an Avatar`);
      }
      console.log(__age); 




      31. If alternates


      1. Ternary Operator: condition ? trueValue : falseValue
                  Quick one-line if-else.
                  
      2. Guard Operator: value || defaultValue
                  Use when a fallback value is needed.

      3. Default Operator: value ?? fallbackValue
                  Use when you want to consider only null and undefined as falsy.

      4. Simplifies conditional logic.

      5. Use wisely to maintain readability.

      Example:-- 

      
    console.log('\n\n31. If - alternatives\n');

    console.log('Ternary Operator\n\n');

    // let $$age = 21;
    // let $$result;
    // if($$age > 18){
    //   $$result = 'Adult';
    // } else {
    //   $$result = 'Kid';
    // }
    // console.log($$result);


    //     condition         ? trueValue : falseValue;
    let $$age;
    let $$result = $$age > 18  ?  'Adult'  :   'Kid';

    console.log('\n\nGuard Operator');
    // Whevever there's Falsy value, default will take it's place.              

    let _$age ;
    //             value || defaultValue
    let finalAge = _$age || 18;

    console.log(_$age);
    console.log(finalAge);

    console.log('\n\nDefault Operature\n')
    // Whevever there's Falsy value EXCEPT null and undefined, fallback will take it's place.  

    let $_age ;
    //             value  ?? fallbackValue
    let final_Age = _$age ??   18;

    console.log($_age);
    console.log(final_Age);
