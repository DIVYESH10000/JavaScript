      36. What is an Object

                               
      
      

      1. Groups multiple values together in key-value pairs.

      2. How to Define: Use {} to enclose properties.

      3. Example: product {name, price}.
      
      4. Dot Notation: Use . operator to access values.

      5. Key Benefit: Organizes related data under a single name.

      Example: ---


      37. Objects Syntax

      


      1. Basic Structure: Uses {} to enclose data.
      2. Rules: Property and value separated by a colon(:).
      3. Comma: Separates different property-value pairs.
      4. Example: { name: "Laptop", price: 1000 }.

      
      38. Accessing Objects

      1. Dot Notation: Access properties using . Operatorlike product.price

      2. Bracket Notation : Useful for properties with special characters product[”nick-name"]. Variables can be used to access properties.

      4. Values can be added or removed to an object.

      5. Delete Values using delete.

      39. Inside Objects

               



            
              1. Objects can contain Primitives like numbers and strings.

              2. Objects can contain other objects and are called Nested Objects.

              3. Functions can be object properties.

              4. Functions inside an object are called methods.
              
              5. Null Value: Intentionally leaving a property empty.

              Note : When a function is inside an Object, it's called a Method.
            


            Example: 

            
              console.log('\n\n39. Inside Objects\n');

              let newProduct = {
                newCompany : 'Fendi',
                'new-Item' : 'T-Shirt',
                price : 9999,
  
                              // Anonymous Function
                displayPrice: function() {
                  // console.log(`Price of the product is : ₹${newProduct.price}`)   
                  // As its's happening inside the same object, we won't know the object name so we'll use:
                    
                  this.price ( implies CURRENT OBJECT)
  
                  console.log(`Price of the product is : ₹${this.price}`) 
                }
              };
      
              newProduct.displayPrice();
            

    // console.log() : Means console is an Object and inside it there's a method log()

    // Math.random() : Means Math is an Object and inside it there's a method random()


      40. Autoboxing

      

      1. Automatic conversion of primitives to objects.

      2. Allows properties and methods to be used on primitives.

      3. Example: Strings have properties and methods like length, toUpperCase, etc.

      Example: 
      // 40. Autoboxing
        console.log('\n\n40. Autoboxing\n');
        console.log('On a dark Desert Highway, Cool wind in my hair'.toUpperCase());
        console.log('I am the best'.replace('best', 'GREATEST'));
        console.log('JavaScript is Awesome'.charAt(0));
        console.log('JavaScript is Awesome');

      41. Object References

          



      1. Objects work based on references, not actual data.

      2. Copying an object copies the reference, not the actual object.
      
      3. When comparing with ==, you're comparing references, not content.
    
      4. Changes to one reference affects all copies.


      Examples : 

      // 41. Object References


   
    // Pass by Value
    console.log('\n\n41. Object References\n');

    console.log('\n\nPass by Value\n');

    let a = 5;
    let b = a;  // new variable b is created at different MEMORY ADDRESS

    console.log(`let a = 5;\nlet b = a;`);

    console.log(`Value of a : ${a}\nValue of b : ${b} `);

    a = 8;
    console.log('a = 8');
    console.log(`Value of a : ${a}\nValue of b : ${b} `);
   

    
      //  Pass by Reference
    console.log('\n\nPass by Reference\n\n');
    let x = {num : 5};  // object x at        Address xxx1234
    
    let y = x;  // Here new object is not created, ONLY reference of  {num : 5} is given to y.
                 // y will also point to same address.

    console.log(`let x = {num = 5};\nlet y = x;`);

    console.log(`Value of x.num : ${x.num}\nValue of y.num : ${y.num}`);

    y.num = 99;

    console.log(`Value of x : ${x.num}\nValue of y : ${y.num}`);
    console.log(`When we change value of y.num, value of x.num also changes.\nThis means both References i.e. ref of x and ref of y\nWERE POINTING TO same Address Location.`);

    

    // When comparing with ==, you're comparing references, not content.
    console.log(`When comparing with ==, you're comparing references, not content`);
    
    let p = {pop: 'abcd'};
    let q = {pop: 'abcd'};

    console.log( p == q);     // false
    console.log( p === q);     // false

    console.log('Both case are false BECAUSE \nwhile comparing OBJECTS ==, === makes no difference. ONLY REFERENCES are compared, not the deep values(gotta do that individually from comparing individual values).');

      42. Object Shortcuts

      


      1. De-structuring: Extract properties from objects easily.

      2. We can extract more than one property at once.

      3. Shorthand Property:  {message: message} simplifies to just message.

      4. Shorthand Method: Define methods directly inside the object without the function keyword.
